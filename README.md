# Data Science & Analytics Labs

Welcome to the **Data Science & Analytics Labs** repository. This repository contains a structured collection of Jupyter notebooks, datasets, and visualizations covering core data science methodologies, exploratory data analysis (EDA), database concepts, and data manipulation techniques across multiple weeks.

---

## 📂 Repository Structure

The repository is organized chronologically by week and laboratory topic:

### 🗓️ Week 1: Introduction to Data Analysis
Focuses on establishing foundational skills in data ingestion, exploration, and initial handling of weather-related datasets.

*   **Lab 1:** `Week 1 lab 1.ipynb` — Introductory data exploration notebook.
*   **Lab 2:** `week 1 lab 2 (1).ipynb` — Core concepts of data manipulation.
*   **Lab 3 & 4:** `week_1_lab_3 , 4.ipynb` — Advanced analysis focusing on time-series weather conditions.
*   **Datasets:**
    *   `hourly_weather_10_days.csv`
    *   `weather_data.csv`

### 🗓️ Week 2: Advanced Pandas & Numerical Computing
Focuses heavily on relational data operations, pivot operations, handling missing values, and scientific computing with NumPy.

*   **Lab 1: Dataframe Mastery & Relational Operations**
    *   `dataframe_basics.ipynb` — Foundational DataFrame manipulations.
    *   `handling_missing_data_replace.ipynb` — Strategy and execution for cleaning null/missing data.
    *   `pandas_concat.ipynb` — Structural stacking and concatenation.
    *   `pandas_merge.ipynb` — Relational joins and database-style merging.
    *   `pandas_group_by.ipynb` — Split-apply-combine aggregation operations.
    *   `pandas_pivot.ipynb` — Reshaping data matrices using pivot tables.
*   **Lab 2: Scientific Computing & Visualization**
    *   `Numpy_and_Plotting (1).ipynb` — Vectorized math operations with NumPy and data plotting.
*   **Visual Assets & Schematics:**
    *   `db_joins.jpg` — Diagram mapping database join types.
    *   `group_by_cities.png` — Visualization of geographic grouping outputs.
*   **Reference Datasets:**
    *   `weather.csv` | `weather2.csv` | `weather3.csv` 
    *   `weather_by_cities.csv` | `weather_data.csv`
